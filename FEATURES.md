# Codo 功能规划详细文档

## 一、网页内容总结

### 触发方式
- 用户通过 Telegram Bot 发送 URL
- 用户通过 PWA 网页提交 URL

### 处理流程
1. 收到 URL，写入任务队列，状态 `pending`
2. Worker 判断页面类型：
   - 静态页：直接 HTTP 请求抓取正文
   - JS 渲染 / 懒加载：Playwright 无头浏览器，模拟滚动等待加载
   - CF 防护：转发给 Flaresolverr 处理后返回内容
3. 提取正文（去除导航、广告、页脚等噪音）
4. 调用 Claude API 生成摘要（300字以内）
5. 存入知识库（标题、URL、摘要、原文、时间、标签）
6. 推送摘要给用户

### 输出格式
```
📄 文章标题
来源：example.com

摘要：
[300字以内的内容总结]

🔗 原文链接
```

## 二、视频内容总结

### 支持平台
- YouTube（通过字幕 API 获取文字稿）
- B站（通过 AI 字幕接口）

### 处理流程
1. 收到视频 URL，识别平台
2. 获取视频字幕 / 文字稿
3. 字幕过长时分段处理，Claude API 逐段总结后合并
4. 生成视频内容摘要
5. 存入知识库，推送用户

### 输出格式
```
🎬 视频标题
时长：XX 分钟  来源：YouTube

核心内容：
[视频主要讲了什么]

关键点：
- 要点一
- 要点二
- 要点三
```

## 三、微信公众号星标订阅

### 方案
自建 RSSHub，配置微信凭证，订阅星标公众号的 RSS feed。

### 处理流程
1. 用户配置公众号 ID 列表（或通过抓包导入）
2. Codo 每小时拉取一次各公众号的 RSS
3. 对比数据库，发现新文章
4. 抓取文章正文，调用 Claude API 生成摘要
5. 存入知识库，推送用户

### 凭证过期处理
凭证失效时，Codo 推送提醒："微信公众号凭证已过期，请重新配置"，用户重新抓包更新即可。

## 四、RSS 聚合

### 处理流程
1. 用户添加 RSS 订阅源 URL
2. 每小时定时拉取，与数据库去重
3. 新条目调用 Claude API 生成摘要
4. 按订阅源分组，每日汇总推送一次日报

### 日报格式
```
📰 今日信息日报 - 2026/06/04

【前端动态】
- [Vite 6.1 发布] 新增 XXX 特性...
- [Vue 3.5 更新] 优化了 XXX...

【AI 资讯】
- [Claude 新功能] ...

共 12 条新内容，完整内容见知识库。
```

## 五、学习通作业提醒

### 处理流程
1. 用户首次授权登录学习通（Playwright 扫码）
2. 保存登录态，每天定时检测作业列表
3. 发现新作业：立即推送通知
4. 作业截止前 24 小时：再次提醒

### 推送格式
```
📚 学习通提醒

新作业：《数据结构》第三章作业
截止时间：2026/06/07 23:59（还有 3 天）

⚠️ 即将截止：《操作系统》实验报告
截止时间：明天 23:59（还有 1 天）
```

## 六、linux.do 收藏同步

### 处理流程
1. 用户授权登录 linux.do（Playwright）
2. 每小时同步一次收藏列表
3. 发现新收藏，抓取帖子正文
4. Claude API 生成摘要，存入知识库
5. 可选：推送通知 或 静默入库

---

## 七、邮件助理

### 支持协议
- IMAP（Gmail、QQ邮箱、企业邮箱）

### 处理流程
1. 用户配置邮箱 IMAP 授权
2. 定时拉取未读邮件
3. 按重要性分类（账单、通知、重要、垃圾）
4. 重要邮件生成摘要推送
5. 每日汇总：今日 X 封邮件，X 封重要

---

## 八、个人知识库

### 存储结构
```
文章
  - id, url, title, source
  - content（原文）
  - summary（摘要）
  - embedding（向量，用于语义检索）
  - tags（自动打标签）
  - created_at
```

### 查询方式
- 关键词搜索
- 语义搜索（pgvector 向量检索）
- 自然语言提问（"我收藏过关于 Go 并发的文章吗"）

---

## 九、实时任务看板

### 页面内容
- 当前执行中的任务（实时进度）
- 今日已完成任务列表
- 失败任务（含原因 + 重试按钮）
- 知识库统计（总文章数、今日新增）

### 技术实现
- 后端通过 WebSocket 推送任务状态变更
- 前端实时更新，无需手动刷新

## 十、工作助理

### 日程提醒
- 用户发自然语言："明天下午3点提醒我开会"
- Claude 解析时间和内容，存入提醒表
- 到时间通过 Telegram 推送

### 资料检索
- 用户提问，先检索本地知识库
- 知识库无结果时，调用网络搜索补充
- 整合结果给出答案，附来源

### 智能翻译
- 发 URL → 抓取正文 → 翻译成中文 → 推送
- 发文字 → 直接翻译

## 功能优先级

| 优先级 | 功能 | 说明 |
|--------|------|------|
| P0 | 网页内容总结 | 核心主链路，第一期必做 |
| P0 | Telegram Bot 接入 | 主要输入和通知渠道 |
| P0 | 任务看板 | 可观测性，必须有 |
| P0 | 个人知识库 | 内容沉淀，核心价值 |
| P1 | RSS 聚合 | 主动订阅场景 |
| P1 | 微信公众号订阅 | 国内用户核心需求 |
| P1 | 学习通提醒 | 刚需，实现不复杂 |
| P2 | 视频内容总结 | 有价值但实现较复杂 |
| P2 | 邮件助理 | 补充场景 |
| P2 | linux.do 同步 | 补充场景 |
| P3 | 工作助理（日程/翻译） | 锦上添花 |
